(c) What is the difference between git log and git status?
git status displays the state of the working directory and staging area. It allows users to see which changes have been staged, which have not been staged and which files are not being tracked by git

git log displays project history and allows the user to filter and search the history.  

(d) What is the difference between unstaged and untracked files?
Unstaged files: tracked but not has not been staged

Untracked files are files whose changes are not tracked by git

(e) What is the difference between git checkout and git reset?
git checkout replaces the current state with the state of the requested commit

git reset resets the current pointer to the given commit

(f) What happens if you try to pull remote changes when there are uncommitted
changes in your working directory?
